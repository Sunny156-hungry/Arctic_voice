Data governance in this project is directly embedded in the design and development of the Arctic Voice website. Transparency is ensured by clearly documenting every dataset used, including the China Cetacean Alliance (CCA) report attachments and any supplementary conservation references. All raw XLSX files are stored in the /CCA_tables/ folder, the cleaning script is available in /scripts/data_cleaning.py, and the processed unified dataset is saved in /data/cca_unified_dataset.csv. [7] The README explains data provenance, processing steps, and known limitations. GitHub commit history provides an open record of revisions and updates. [3]
Privacy considerations are straightforward but important. The project does not collect user data. Incident records from CCA are presented at the facility and species level rather than identifying individual staff or private persons. Narrative components such as Sophie’s Story are clearly marked as interpretive storytelling, separated from verified data. [9]
Inclusivity is reflected in the integration of investigative NGO data with broader conservation and policy context, avoiding a single institutional viewpoint. Accessibility is addressed through machine-readable formats (CSV, JSON), bilingual labeling (Chinese–English species names), and clear visual encodings designed for museum visitors and student learners.
